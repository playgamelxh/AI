要求简介：
在AI/baokao目录下，实现一个帮助考生高考填志愿的项目，考生通过该项目能查询选中高校在当前省份历年录取人数，最低分等；系统可以根据考生填写的分数和兴趣爱好等信息，推荐适合的大学和专业。

项目要求：
1：Golang语言实现gin框架，使用1.25版本golang代码
2：MySQL存储相关数据，使用migrate管理数据库迭代
3：前端用饿了么开源框架，vue框架
4：使用docker-compose管理，docker部署

项目细节：
1：前端展示
学校展示
展示高校基本信息列表，并根据985、211、双一流、本科、专科，军校、艺术类等设置标签，方便考生区分，查看和筛选
展示高校详情，要尽可能详细展示高校信息，包括不限于，高校名称、简介、详情、网址、招生办电话等等。
考生咨询
要求考生提供考试成绩、兴趣爱好，喜好城市、喜欢专业等，擅长学科，通过调用AI大模型给考生提供尽可能详细的报考计划，比如给与5个一批志愿，5个平行志愿等等。

2：后端管理
1. 学校管理功能
学校基本信息管理
需要将中国各批次高等院校按分类 提前批、艺术、985、211、双一流、本、专科等分类管理，学校信息尽可能全面完整，并提供管理接口，能够支持新增、编辑、删除、查看详情等，实现高校搜索列表，支持按城市，按分类搜索高校。
学校专业管理
管理每个学校的专业设置，以当前最新的专业设置为准，提供增、删、改、查，列表展示接口，
学校各专业在各省历年招收最低分管理
设置学校专业，在各省历年招生最低分，提供增、删、改、查，列表展示接口
2. 省份管理
需要将中国所有省份都填充上去，并包含省份编号，包含港澳台，

## 版本迭代记录

### v1.0.0 第一版迭代 - 项目初始化与省份管理模块
**更新日期:** 2026-03-31
**更新内容:**
1. **项目基础架构初始化**
   - 初始化基于 Golang 1.25 和 Gin 框架的后端项目，并集成 MySQL 驱动及 golang-migrate 库，提供自动化的数据库迁移。
   - 初始化基于 Vue 3、Vite 和 Element Plus 的前端管理后台项目，并配置 axios 以及 vue-router。
   - 编写 `docker-compose.yml` 及前后端 Dockerfile，实现了数据库、后端 API、前端静态服务的一键容器化编排。
2. **后端功能**
   - 设计并实现 `provinces` 数据表结构，并提供 `migrate` up/down 脚本，其中默认包含中国所有省份（含港澳台）的基础数据。
   - 实现省份信息的 CRUD 接口，支持省份的查询、添加、修改、删除等操作，支持跨域请求。
3. **前端功能**
   - 搭建管理后台的基本布局（侧边栏、顶部导航），使用 Element Plus 提供现代化的 UI 展示。
   - 实现“省份管理”页面，支持在表格中展示省份列表，并通过对话框实现新增和编辑省份信息，结合 Axios 与后端进行数据交互。

### v1.1.0 第二版迭代 - 核心业务数据表设计与 MVC 架构重构
**更新日期:** 2026-03-31
**更新内容:**
1. **数据库结构扩展**
   - 保证所有表结构包含主键 `id` 字段。
   - 新增 `schools` (学校表)，涵盖省份ID、城市、办学层次（本科/专科）、985/211/双一流标签、类别、批次及详细介绍等。
   - 新增 `majors` (学校专业表)，关联学校表，管理专业基本信息。
   - 新增 `major_scores` (学校专业各省最低分表)，增加 `year` (年份) 维度以及录取批次、科类和录取人数，关联学校、专业、省份等维度。
2. **后端 MVC 架构重构**
   - 将原有的平铺式 API 设计重构为标准的 MVC 模式，提升代码可维护性。
   - `Router` 层 (`internal/router`)：统一注册所有业务模块（省份、学校、专业、分数）的路由，并独立于入口文件。
   - `Controller` 层 (`internal/controller`)：仅负责 HTTP 请求参数解析、入参校验，调用 Service 层获取结果并封装返回响应。
   - `Service` 层 (`internal/service`)：封装详细的业务逻辑和处理过程，供 Controller 层调用。
   - `Model` 层 (`internal/model`)：定义各业务实体（学校、专业、省份、分数）对应的结构体，并实现与 MySQL 的底层交互和基础的增删改查方法。
3. **接口扩展**
   - 实现并支持了学校的按城市、类别搜索列表和基础的增删改查接口。
   - 实现并支持了学校专业的增删改查列表接口。
   - 实现并支持了学校专业各省历年招收最低分的增删改查列表接口。

### v1.2.0 第三版迭代 - 数据初始化填充与AI咨询功能框架
**更新日期:** 2026-03-31
**更新内容:**
1. **测试与初始化数据填充**
   - 新增 `migrate` 脚本 `000003_seed_schools_and_scores.up.sql`，预先填充了清华、北大、复旦、上交、浙大及深职大等具有代表性的本专科高校信息。
   - 初始化了各大高校的计算机、汉语言文学、临床医学等典型招生专业。
   - 初始化了 2024、2025 年度上述高校专业在部分省份（如北京、广东、浙江）的最低录取分数线及录取人数，涵盖了物理类、历史类及综合改革等科类。
2. **AI 咨询接口实现**
   - 增加 `/api/ai/consult` 接口框架，用于处理前端传来的考生咨询数据（分数、省份、偏好城市/专业等）。
   - 在 `Service` 层实现 `GenerateAIPlan` 方法，根据考生分数进行基础分层，并结合专业及城市意向，Mock 了一套分批次（冲刺批、平行保底批）的院校志愿推荐逻辑及报考建议，为后续接入真实大模型铺平道路。
3. **前端功能完善**
   - 新增高校管理页面 (`SchoolView.vue`)，实现了高校信息的表格展示。
   - 实现了前端基于城市、类别的组合检索功能，并针对 985、211、双一流 等属性使用了标签（Tag）直观展示。

### v1.3.0 第四版迭代 - 前端管理平台完善与 AI 咨询交互落地
**更新日期:** 2026-03-31
**更新内容:**
1. **前端高校详情功能**
   - 补全了 `README.md` 中要求的“展示高校详情”的功能点，在前端 `SchoolView.vue` 增加了 `详情` 按钮和 `el-dialog` 弹窗。

2. **专业管理与历年分数线查询**

   - 将 AI 返回的报考方案以友好的排版（分为总结、冲刺批次、平行保底批次及报考建议）呈现给考生。
4. **前端路由与导航完善**
